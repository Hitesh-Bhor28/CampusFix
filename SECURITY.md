# Security Policy

## Reporting a Vulnerability
Please report security issues privately and do not open public GitHub issues, discussions, or pull requests for vulnerabilities.

When reporting, please include:
- A clear description of the issue and potential impact
- Steps to reproduce (proof-of-concept if possible)
- Affected versions or components
- Any logs or screenshots that help validate the issue

We aim to acknowledge reports within 72 hours and provide a remediation plan as soon as possible.

## Supported Versions
This project is currently in active development. Security fixes are applied to the latest version only.
